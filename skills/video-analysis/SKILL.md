# 视频内容分析

## Description
分析流媒体视频或本地视频文件的场景结构、语音内容和画面字幕。适用于 YouTube/Bilibili 等视频网站的内容分析需求。

## Instructions

当用户请求分析视频时，使用 `video_analyze` 工具。以下是你的操作指南：

### 适用场景
- "分析这个视频：[URL]"
- "帮我把这个视频转成文字"
- "提取这个视频的字幕"
- "这个视频讲了什么内容"
- "分析这段视频的场景结构"
- 任何涉及视频内容分析的请求

### 使用步骤

**第一步**: 直接调用 `video_analyze` 工具

对于在线视频 URL：
```
{
  "name": "video_analyze",
  "arguments": {
    "url": "用户提供的视频URL"
  }
}
```

对于本地视频文件：
```
{
  "name": "video_analyze",
  "arguments": {
    "file_path": "视频文件路径"
  }
}
```

**第二步**: 将工具的返回结果整理成清晰的回答给用户

### 注意事项

1. **分析时长**: 10 分钟的视频通常需要 3-5 分钟分析。Whisper 语音转文字是最耗时的步骤，如果用户想快速查看结果，可以设置 `skip_transcribe: true`

2. **依赖提示**: 如果工具返回依赖缺失错误，请明确告诉用户需要安装：
   ```
   pip install yt-dlp opencv-python pyscenedetect easyocr faster-whisper pillow numpy
   ```
   以及系统安装 ffmpeg

3. **场景检测阈值**: 默认 30.0。数字越小检测到的场景越多。如果用户觉得场景太少，下次可以调低一些

4. **Whisper 模型**: 默认 base 约 1GB 显存/内存。如果准确度不够可以建议用户下次用 medium 或 large

5. **输出文件说明**:
   - `analysis_report.md` — 可直接阅读的完整报告
   - `analysis_data.json` — 结构化数据，便于程序处理
   - `transcript.srt` — 标准字幕格式，可导入播放器
   - `keyframes/` — 每个场景的关键帧截图

### 示例回答

当工具返回结果后，建议这样组织你的回答：

```
## 🎬 视频分析结果

**标题**: [视频标题]
**时长**: [时长] | **分辨率**: [分辨率]
**场景数**: [N] 个 | **语音转文字**: [N] 条

### 场景概览
[场景时间线摘要]

### 📝 内容要点
[从语音转文字中提炼的关键信息]

📄 完整报告已保存至: [报告路径]
```
